LOCETIUS v2.5 - QUICK START GUIDE
=================================

60-SECOND SETUP
===============

Windows Users:
  1. Extract Locetius_v2.5_production.zip
  2. Install Python 3.10+ from python.org
  3. Open Command Prompt in extracted folder
  4. Run: pip install -r requirements.txt
  5. Run: python locetius_gui_v2.py

Linux Users:
  1. Extract the archive
  2. Run: sudo apt install python3.10 python3-pip
  3. Run: pip install -r requirements.txt
  4. Run: python3 locetius_gui_v2.py

---

YOUR FIRST SOLVE (5 MINUTES)
============================

  1. GUI window opens
  2. Click "Import File" (top left)
  3. Browse to: benchmarks/L_Bracket_Benchmark.mtx
  4. Click "Open"
  5. Confirm: "6,519 variables loaded"
  6. Leave Engine on "AUTO (Smart Route)"
  7. Click the big "Solve with Locetius" button
  8. Watch the green energy curve drop in real-time
  9. When done, you'll see:
     - Solution chart
     - GPU usage stats
     - Consensus percentage
  10. Click "View Solution Graph" to see network visualization
  11. Click "Export To CSV" to save results

Time to completion: ~60 seconds

---

CHOOSING AN ENGINE
==================

AUTO (Smart Route)  -  default, recommended for most users
  Automatically selects LOCETIUS, KERR, or SPARSE based on matrix size
  and GPU memory.

LOCETIUS (Spectral Swarm)  -  classic engine
  Proprietary multi-phase Spectral Swarm algorithm.
  Best for medium problems (N < 50,000).

KERR (Quantum Wave)  -  new in v2.5
  Kerr-nonlinear optical network emulation via CuPy GPU kernels.
  Excellent for dense combinatorial problems.

HYBRID (Blended)  -  new in v2.5
  Fusion of LOCETIUS and KERR engines.
  Use the Hybrid Blend slider to tune the ratio:
    0.0 = pure Kerr  |  0.5 = equal blend  |  1.0 = pure Locetius
  Often gives the best energy of all engines.

SPARSE (Large N)  -  new in v2.5
  cuSPARSE SpMM-based solver for very large sparse problems.
  Auto-activates when N > 10,000 and sparsity > 95%.
  Scales to N ~ 200,000 on 4 GB VRAM.

---

PARALLEL WORLDS (SWARMS)
=========================

64 Worlds  -  default, for 4-8 GB VRAM cards (RTX 2050/3060/4060)
128 Worlds  -  better quality, for 24+ GB VRAM (RTX 3090/4090/A100)

Toggle in the left panel under "Parallel Worlds (Swarms)".

THREE WAYS TO SOLVE PROBLEMS
=============================

Option 1: GUI (User-Friendly) RECOMMENDED
  python locetius_gui_v2.py
  
  Best for: interactive exploration, visual feedback, exports

Option 2: Python SDK (Integration)
  from locetius_api import LOUCETIUSSolver, SwarmConfig
  solver = LOUCETIUSSolver()
  result = solver.solve(Q_matrix, SwarmConfig(
      num_variables=N, num_swarms=64, engine="HYBRID", locetius_weight=0.5
  ))
  
  Best for: embedding in applications, batch solving, custom workflows

Option 3: REST API (Microservices)
  python locetius_server.py
  curl -X POST http://localhost:8765/solve -d '{"Q_dense": [[1, 0.5], [0.5, 2]]}'
  
  Best for: headless deployments, remote solving

WHAT DO THESE DO?
=================

Thermal Melt / Noise (0.0 - 1.0):
  - Lower = More deterministic, faster convergence
  - Higher = More exploration, avoids local minima
  - Default = 0.2

Max Iterations (100 - 50,000):
  - Lower = Faster but lower quality
  - Higher = Slower but better solutions
  - Default = 7,500 (balanced)

Hybrid Blend (0.0 - 1.0):
  - 0.0 = Pure Kerr Wave physics
  - 1.0 = Pure Spectral Swarm physics
  - 0.5 = Equal blend (recommended starting point)

Early Stopping Patience (0 - 5,000):
  - 500 = Good balance (default)
  - 0 = Disabled (run full max iterations)

WHAT DO THESE DO?
=================

Thermal Melt / Noise (0.0 - 1.0):
  - Lower = More deterministic, faster convergence
  - Higher = More exploration, avoids local minima
  - Default = 0.2 (good balance)

Max Iterations (100 - 50,000):
  - Lower = Faster but lower quality
  - Higher = Slower but better solutions
  - Default = 50,000 (thorough search)

Early Stopping Patience (0 - 5,000):
  - If enabled, stops when no improvement for N steps
  - 500 = Good balance (default)
  - 0 = Disabled (run full max iterations)

Phase 3 (Deterministic Freeze):
  - ON = Locks the best solution and polishes it (RECOMMENDED)
  - OFF = Continues full annealing (experimental)

Precision:
  - Float32 = Faster (30-40% speedup), slight accuracy loss
  - Float64 = Slower but highest accuracy (DEFAULT)

INTERPRETING RESULTS
====================

Energy:
  - Lower is better (objective value)
  - Goal: Push energy as negative as possible
  - Graph slope = convergence speed

Consensus:
  - 0-40% = Low agreement (diverse solutions)
  - 40-70% = Moderate agreement
  - 70%+ = High confidence (likely good solution)

Solution States:
  - 1 = Variable is "on" or "selected"
  - 0 = Variable is "off" or "not selected"
  - 0.5 = Undecided (rare, Phase 3 locks these)

Solution Graph:
  - Cyan dots = Variables in state 1
  - Red dots = Variables in state 0
  - Larger dots = Higher consensus
  - Edges = Problem connectivity

GPU Stats:
  - Utilisation > 90% = Good (solver working hard)
  - Temp < 65degC = Healthy, 65-80degC = Normal, >80degC = Hot
  - Power draw = Active computation

EXPORTING RESULTS
=================

Export Solution (.CSV):
  - Simple two-column format: Variable, Value
  - Ready to import into your models
  - Includes consensus percentage

Export Convergence (.JSON):
  - Energy curve over time
  - Perfect for PhD thesis graphs
  - "Time-to-Solution" metrics

Export Topology (.STL):
  - 3D mesh of the solution
  - Import into CAD software (Fusion 360, Blender, etc.)
  - Visualize spatial solutions

LOADING YOUR OWN PROBLEMS
=========================

Matrix Market Format (.MTX):
  * QAPLIB benchmarks
  * GSET benchmark set (MaxCut)
  * Academic research datasets
  * Click "Import File" -> Select .MTX

CSV Edge Format:
  * Columns: Node_A, Node_B, Weight
  * Good for finance/logistics problems
  * Click "Import File" -> Select .CSV

NumPy Format (.npz):
  * Save from Python: np.savez('data.npz', Q=matrix)
  * Load with: locetius_gui opens directly

TROUBLESHOOTING
===============

"CUDA not found":
  -> Update NVIDIA driver from nvidia.com
  -> Ensure driver version 560+

"GUI won't open":
  -> Try: python locetius_gui_v2.py
  -> Check error messages in terminal

"Not using GPU":
  -> Check GPU tab, should show device name
  -> Verify with nvidia-smi in terminal
  -> Try restarting GPU: nvidia-smi -pm 1

"Solve is very slow":
  -> Reduce Max Iterations
  -> Enable Early Stopping
  -> Check GPU utilisation
