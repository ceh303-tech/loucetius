WHAT'S NEW IN v2.5
==================

New Solver Engines
------------------
* KERR Wave Engine  - Quantum-inspired Kerr-nonlinear optical network solver
  Uses CuPy NVRTC GPU kernels simulating parametric oscillator networks.
  Ideal for dense combinatorial problems.
* HYBRID Blend Engine  - Weighted fusion of Spectral Swarm and Kerr Wave
  Adjustable blend slider: 0 = pure Kerr, 1 = pure Locetius (Spectral Swarm).
  Automatically selects the best-energy result.
* SPARSE CUDA Engine  - cuSPARSE SpMM-based solver for very large sparse problems
  Activates automatically for N > 10,000 with sparsity > 95%.
  Scales to N > 200,000 variables on 4 GB VRAM.
* AUTO Smart Route  - Engine selected automatically based on matrix size, density,
  and available GPU memory.

Parallel Worlds (Swarms)
------------------------
* 64-swarm mode: Optimal for consumer GPUs (RTX 20/30/40 series 4-8 GB VRAM)
* 128-swarm mode: New for high-VRAM GPUs (RTX 3090 / 4090 / A100 24+ GB VRAM)
* Selectable at runtime via the Parallel Worlds toggle in the GUI

GUI Enhancements
-----------------
* Engine Selector dropdown (AUTO / LOCETIUS / KERR / HYBRID / SPARSE)
* Parallel Worlds toggle (64 / 128 swarms)  -  live, no restart required
* Hybrid Blend slider (shown only in HYBRID mode): dial from Kerr to Locetius
* Physics Mode section replaces Solver Mode (now clearly LOCETIUS-specific)
* Version badge updated to v2.5

API Changes (locetius_api.py)
------------------------------
* SwarmConfig new fields:
    engine: str = "AUTO"           # "AUTO"|"LOCETIUS"|"KERR"|"HYBRID"|"SPARSE"
    locetius_weight: float = 0.5   # 0.0 = pure Kerr, 1.0 = pure Locetius
    pump_amplitude: float = 0.7    # Kerr pump strength
    kerr_nonlinearity: float = 0.5 # Kerr nonlinear coefficient
* num_swarms: now accepts 64 OR 128 (previously 64 only)

GETTING STARTED
===============

Quick Start (GUI):
  Windows: Double-click RUN_GUI.bat
  Linux: bash RUN_GUI.sh
  -> Load benchmarks/L_Bracket_Benchmark.mtx
  -> Click "Solve with Locetius"
  -> Export results as CSV/JSON/STL

API Example:
  from locetius_api import LOUCETIUSSolver, SwarmConfig
  solver = LOUCETIUSSolver()
  config = SwarmConfig(num_variables=100, annealing_steps=2000)
  result = solver.solve(Q_matrix, config)

REST Example:
  python locetius_server.py  # Start server
  curl -X POST http://localhost:8765/solve -d '{"Q_dense": [[1, 0.5], [0.5, 2]]}'

See README.md and docs/LOCETIUS_USER_MANUAL.docx for full documentation.
